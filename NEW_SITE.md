# Setting up a new jsPsych-ecosystem docs site

This is the complete path from an empty folder to a deployed docs site that
shares the jsPsych family brand, navbar ecosystem switcher, and search. It
assumes the docs live in a `docs/` folder of your project repo (adjust paths
if they live at the repo root) and Node ≥ 20.

## 1. Minimal file set

```
docs/
├── package.json
├── docusaurus.config.ts
├── sidebars.ts
├── tsconfig.json
├── docs/
│   └── intro.md
└── static/
    └── img/            # favicon + any images your pages use
```

**`package.json`** — the version pins and the `overrides` block are load-bearing;
see [Version pins](#version-pins) below before changing them:

```json
{
  "name": "my-tool-docs",
  "version": "0.0.0",
  "private": true,
  "scripts": {
    "docusaurus": "docusaurus",
    "start": "docusaurus start",
    "build": "docusaurus build",
    "serve": "docusaurus serve",
    "clear": "docusaurus clear",
    "typecheck": "tsc"
  },
  "dependencies": {
    "@docusaurus/core": "3.9.2",
    "@docusaurus/preset-classic": "3.9.2",
    "@jspsych/docusaurus-preset": "^0.2.0",
    "@jspsych/docusaurus-theme": "^0.2.0",
    "@mdx-js/react": "^3.0.0",
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "@docusaurus/module-type-aliases": "3.9.2",
    "@docusaurus/tsconfig": "3.9.2",
    "@docusaurus/types": "3.9.2",
    "typescript": "~5.6.2"
  },
  "overrides": {
    "@docusaurus/plugin-content-docs": "3.9.2",
    "webpack": "5.105.3"
  },
  "engines": {
    "node": ">=20.0"
  }
}
```

**`docusaurus.config.ts`** — only per-site facts; everything shared comes from
the factory:

```ts
import {defineJspsychConfig} from '@jspsych/docusaurus-preset';

export default defineJspsychConfig({
  title: 'My Tool',
  tagline: 'One line about what this is',
  url: 'https://jspsych.github.io',
  baseUrl: '/my-tool/',
  organizationName: 'jspsych',
  projectName: 'my-tool',
  favicon: 'img/favicon.png',
  githubUrl: 'https://github.com/jspsych/my-tool',

  navbar: {
    title: 'My Tool',
    items: [
      {type: 'docSidebar', sidebarId: 'docs', position: 'left', label: 'Docs'},
    ],
  },

  docs: {
    sidebarPath: './sidebars.ts',
    editUrl: 'https://github.com/jspsych/my-tool/tree/main/docs/',
  },

  footerLinks: [
    {title: 'Docs', items: [{label: 'Get Started', to: '/docs/intro'}]},
  ],
});
```

The factory automatically adds: brand CSS + fonts, the search field, the
Ecosystem dropdown, the GitHub icon (from `githubUrl`), the branded footer +
copyright, shared Algolia search config, prism themes, and dark-mode behavior.
Every option can be overridden — see `JspsychConfigOptions` in
[`packages/preset/src/defineConfig.ts`](packages/preset/src/defineConfig.ts)
for the full surface, including the `extraConfig` escape hatch.

**`sidebars.ts`**:

```ts
import type {SidebarsConfig} from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  docs: [{type: 'autogenerated', dirName: '.'}],
};

export default sidebars;
```

**`tsconfig.json`**:

```json
{
  "extends": "@docusaurus/tsconfig",
  "compilerOptions": {"baseUrl": "."},
  "exclude": [".docusaurus", "build"]
}
```

**`docs/intro.md`** — any markdown. Then:

```bash
npm install
npm start        # dev server with the full brand applied
```

## 2. Shared components in your pages

- **Runnable experiment listings**: tag a fence with `runnable` and a
  self-contained HTML document — readers get an "open & run in new tab"
  button next to copy/wrap:

  ````md
  ```html runnable
  <!DOCTYPE html>
  <html> ... a complete jsPsych experiment ... </html>
  ```
  ````

- **Live editable plugin demos** (MDX):

  ```mdx
  import {PluginExample} from '@jspsych/docusaurus-theme/components';

  <PluginExample title="Demo" code={`const trial = {...};\nconst timeline = [trial];`} />
  ```

  Defaults (jsPsych version, controls plugin, harness paths) come from the
  theme; override per-site via the `jspsych.pluginExample` option of
  `defineJspsychConfig`, or per-instance via props.

- **Citation cards** (MDX): `import {Citation} from '@jspsych/docusaurus-theme/components';`

The example pages in [`examples/site/docs/`](examples/site/docs/) exercise all
of these and are safe to copy from.

## 3. Deploy to GitHub Pages

1. Copy [`templates/publish-docs.yml`](templates/publish-docs.yml) to
   `.github/workflows/publish-docs.yml` in your repo; adjust `working-directory`,
   the `paths` filter, and `cache-dependency-path` if your site isn't in `./docs`.
2. In the repo settings: **Settings → Pages → Source: GitHub Actions**.
3. Push to `main`. The site appears at `https://<org>.github.io/<repo>/` —
   which must match the `url` + `baseUrl` you set in the config (the runnable
   code-fence button rewrites asset paths against them).

## 4. Join the ecosystem

Open a PR against **this repo** that:

1. Adds your site to
   [`packages/theme/src/sites.ts`](packages/theme/src/sites.ts) — this puts it
   in the Ecosystem navbar dropdown of every family site.
2. Adds your site's URL to [`crawler/config.json`](crawler/config.json) with a
   `site:<id>` tag — this includes it in the shared search index.

Include a changeset (`npx changeset`, patch bump for the theme). Once the
release PR merges and publishes, every family site picks up your entry on its
next rebuild.

## Version pins

Two pins in the `overrides` block keep fresh installs working and should be
copied verbatim:

- `webpack: 5.105.3` — webpack ≥ 5.106 rejects the options webpackbar 6 (used
  by Docusaurus 3.9) passes to ProgressPlugin, which kills the production build.
- `@docusaurus/plugin-content-docs: 3.9.2` — a wildcard peer dependency inside
  preset-classic otherwise resolves to a second, newer Docusaurus core in the
  same tree, with the same webpackbar breakage.

Everything `@docusaurus/*` is pinned to exactly `3.9.2` because the theme's
swizzled components track theme-classic 3.9.x internals. When the theme moves
to a newer Docusaurus, that ships as a theme release with updated guidance
here — don't upgrade the Docusaurus packages in your site independently.
